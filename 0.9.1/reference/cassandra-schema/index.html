    Cassandra Schema Reference¶

    WitFoo Analytics uses Apache Cassandra 5.0+ with Storage Attached Indexing (SAI) as its primary data store. All tables reside in the analytics_v2 keyspace.

    CREATE KEYSPACE IF NOT EXISTS analytics_v2
WITH replication = {'class': 'SimpleStrategy', 'replication_factor': 1}
AND durable_writes = true;

    SAI Indexing

    All secondary indexes use Cassandra 5.0 Storage Attached Indexing (StorageAttachedIndex). SAI provides efficient filtering without the overhead of traditional secondary indexes. Map columns support both VALUES() and ENTRIES() index types.

    artifacts¶

    Core signal/artifact storage. Uses map-based columns for flexible field indexing.

    Compaction: TimeWindowCompactionStrategy (6-hour windows)

    Column Type Description
    partition_key text Tenant/org identifier
    time_bucket text YYYY-MM-DD-HH time bucket
    created_at timeuuid Creation time (clustering, DESC)
    artifact_id uuid Unique artifact identifier
    tags set<text> Classification tags
    artifact_json text Full artifact JSON (backward compat)
    created_at_time timestamp Creation timestamp
    updated_at timestamp Last update timestamp
    artifact_numbers map<text, double> Numeric fields (severity, ports, bytes)
    artifact_ipv4 map<text, inet> IPv4 address fields
    artifact_ipv6 map<text, inet> IPv6 address fields
    artifact_text map<text, text> All text fields (message_type, action, etc.)
    matched_lead_rules frozen<list<int>> Matched lead rule IDs
    matched_lead_rules_updated_at timestamp Lead rules match timestamp
    stix_indicator_ids frozen<list<text>> STIX indicator references
    attack_technique_ids frozen<list<text>> MITRE ATT&CK technique IDs
    attack_tactic_ids frozen<list<text>> MITRE ATT&CK tactic IDs
    attack_detection_strategy_ids frozen<list<text>> Detection strategy IDs
    node_index map<text, text> Direct node_id references for graph enrichment
    is_alarm boolean Denormalized alarm flag
    severity_level int Denormalized severity for range queries

    Primary Key: ((partition_key, time_bucket), created_at, artifact_id)

    SAI Indexes: artifact_id, tags, VALUES(artifact_numbers), VALUES(artifact_ipv4), VALUES(artifact_ipv6), VALUES(artifact_text), ENTRIES(artifact_text), ENTRIES(artifact_ipv4), ENTRIES(artifact_numbers), matched_lead_rules, matched_lead_rules_updated_at, stix_indicator_ids, attack_technique_ids, is_alarm, severity_level

    objects¶

    Generic object storage for licensing metadata, products, and frameworks.

    Compaction: UnifiedCompactionStrategy

    Column Type Description
    partition_key text Partition identifier
    created_at timeuuid Creation time (clustering, DESC)
    object_type text Object type discriminator
    object_id uuid Unique object identifier
    object_json text Full object JSON

    Primary Key: (partition_key, created_at, object_id)

    SAI Indexes: object_type (case-insensitive)

    graph_nodes¶

    Graph entity nodes representing assets, users, files, processes, domains, and URLs.

    Compaction: TimeWindowCompactionStrategy (1-day windows)

    Column Type Description
    partition_key text Org ID for multi-tenancy
    node_id timeuuid UUID v1 for time-ordering (clustering, DESC)
    node_type text Entity type: asset, user, file, process, domain, url
    node_numbers map<text, double> Numeric fields
    node_text map<text, text> Text fields (ip_address, hostname, etc.)
    node_flags map<text, boolean> Boolean fields (internal, managed, trusted)
    node_times map<text, timestamp> Timestamp fields
    sets frozen<set<int>> Set memberships
    products frozen<set<int>> Product associations
    frameworks frozen<set<int>> Framework tags
    products_updated_at timestamp Products last updated
    frameworks_updated_at timestamp Frameworks last updated
    matched_lead_rules frozen<list<int>> Matched lead rule IDs
    vulns frozen<set<text>> CVE identifiers
    last_vuln_scan timestamp Last vulnerability scan
    created_at timestamp Node creation time
    updated_at timestamp Last update time
    observed_at timestamp Last observation time
    metadata text Additional JSON metadata

    Primary Key: (partition_key, node_id)

    SAI Indexes: VALUES(node_numbers), VALUES(node_text), VALUES(node_flags), VALUES(node_times), node_type, products_updated_at, frameworks_updated_at

    node_index¶

    O(1) lookup table mapping identifiers (IP addresses, hostnames, etc.) to node IDs.

    Column Type Description
    partition_key text Org ID
    index_type text Identifier type: ip, hostname, credential, etc.
    index_value text Normalized identifier value
    node_id timeuuid References graph_nodes.node_id
    node_type text Denormalized entity type
    updated_at timestamp Last update time

    Primary Key: (partition_key, index_type, index_value)

    SAI Indexes: node_id

    graph_edges¶

    Relationships between graph nodes (connections, sessions, presence, attribution).

    Compaction: TimeWindowCompactionStrategy (6-hour windows)

    Column Type Description
    partition_key text Org ID
    edge_id text Composite: {source_id}-{target_id}-{type}-{subtype}
    source_id timeuuid Source node UUID
    target_id timeuuid Target node UUID
    edge_type text connection, session, presence, attribution
    edge_subtype int Type-specific subtype
    edge_numbers map<text, double> Numeric fields
    edge_text map<text, text> Text fields
    edge_times map<text, timestamp> Timestamp fields
    bytes bigint Data transfer volume
    count bigint Occurrence count
    artifacts frozen<list<timeuuid>> Evidence trail (last 100 artifacts)
    matched_lead_rules frozen<set<int>> Matched lead rule IDs
    mo_id int MO definition ID
    incident_ids frozen<set<timeuuid>> Associated incident IDs
    started_at timestamp First observation
    ended_at timestamp Last observation
    created_at timestamp Creation time
    updated_at timestamp Last update time
    metadata text Additional JSON metadata

    Primary Key: (partition_key, edge_id)

    SAI Indexes: VALUES(edge_numbers), VALUES(edge_text), VALUES(edge_times), source_id, target_id, edge_type, matched_lead_rules, mo_id, incident_ids

    lead_rules¶

    Global pattern matching rule definitions. Partitioned by category for efficient loading.

    Compaction: LeveledCompactionStrategy

    Column Type Description
    rule_category text Partition: artifact, node, edge, global
    rule_id int Unique rule identifier (clustering, ASC)
    description text Rule description
    rule_type int 1=artifact, 2=node, 3=edge
    enabled boolean Whether rule is active
    criteria text Match criteria expression
    suspicion_boost double Suspicion score contribution (0.0–1.0)
    client_set_id int Client behavioral set reference
    server_set_id int Server behavioral set reference
    file_set_id int File behavioral set reference
    user_set_id int User behavioral set reference
    service_set_id int Service behavioral set reference
    hardware_set_id int Hardware behavioral set reference
    foreign_id int External reference ID
    product_id int Product association
    criteria_type int Criteria type discriminator
    rule_source int Rule source identifier
    created_at timestamp Creation time
    updated_at timestamp Last update time

    Primary Key: (rule_category, rule_id)

    SAI Indexes: enabled, rule_type

    leads¶

    Pattern match results linking rules to entities.

    Compaction: TimeWindowCompactionStrategy (1-day windows)

    Column Type Description
    org_id text Organization identifier
    lead_id timeuuid Unique lead ID (clustering, DESC)
    rule_id int Matched rule reference
    entity_type text artifact, node, or edge
    entity_id uuid Matched entity ID
    matched_at timestamp Match timestamp
    suspicion_contribution double Score contribution
    metadata text JSON match details
    incident_id timeuuid Assigned incident
    is_block_action boolean Block/deny action flag

    Primary Key: (org_id, lead_id)

    SAI Indexes: entity_id, rule_id, incident_id, entity_type

    incidents_v2¶

    Aggregated security incidents with monthly time-bucket partitioning.

    Compaction: TimeWindowCompactionStrategy (7-day windows)

    Column Type Description
    org_id text Organization identifier
    time_bucket text YYYY-MM monthly partition
    incident_id timeuuid Unique incident ID (clustering, DESC)
    incident_name text Generated memorable name
    parent_incident_id uuid Parent incident for splits
    part_number int Split part number
    total_parts int Total split parts
    title text Incident title
    description text Incident description
    severity_level int 1=low, 2=medium, 3=high, 4=critical
    status_id int 1=new, 2=investigating, 3=resolved, 4=closed
    assigned_to text Primary assignee
    source_ips set<text> Source IP addresses
    dest_ips set<text> Destination IP addresses
    users set<text> Associated users
    tags int MO definition ID
    mo_name text Denormalized MO name
    suspicion_score double Aggregate suspicion (0.0–1.0)
    observation_results map<int, text> Observation ID → JSON result
    artifact_count int Number of artifacts
    lead_count int Number of leads
    graph_json text Cytoscape visualization JSON
    metadata text Additional JSON metadata
    created_at timestamp Creation time
    updated_at timestamp Last update time

    Primary Key: ((org_id, time_bucket), incident_id)

    SAI Indexes: assigned_to, source_ips, dest_ips, users, tags, suspicion_score, node_ids, status_id, severity_level, mo_name

    roles¶

    RBAC role definitions with permission lists.

    Column Type Description
    role_id uuid Unique role identifier
    org_id text Organization scope
    role_name text Display name
    description text Role description
    permissions frozen<list<text>> Assigned permission keys
    system_role boolean Built-in role flag (cannot delete)
    created_at timestamp Creation time
    updated_at timestamp Last update time
    created_by uuid Creator user ID
    updated_by uuid Last updater user ID

    Primary Key: (org_id, role_id)

    SAI Indexes: role_name (case-insensitive), system_role

    Predefined Roles

    See Roles Reference for the 8 system roles seeded on initialization.

    users¶

    User account storage for authentication and authorization.

    Column Type Description
    user_id uuid Primary key
    username text Login username
    email text Email address
    password_hash text Bcrypt password hash
    first_name text First name
    last_name text Last name
    role_ids frozen<list<uuid>> Assigned role references
    org_id text Organization scope
    active boolean Account active flag
    is_platform_admin boolean Cross-tenant super admin flag
    last_login timestamp Last login timestamp
    created_at timestamp Creation time
    updated_at timestamp Last update time
    created_by uuid Creator user ID
    updated_by uuid Last updater user ID
    locale text User locale preference

    Primary Key: (user_id)

    SAI Indexes: username (case-insensitive), email (case-insensitive), org_id, active, is_platform_admin

    settings¶

    Organization-level configuration. One row per organization.

    Column Type Description
    org_id text Primary key — organization identifier
    org_name text Organization name
    org_display_name text Display name
    org_domain text Organization domain
    timezone text Timezone setting
    date_format text Date format preference
    severity_threshold int Minimum severity for alerts
    auto_close boolean Auto-close incidents
    auto_close_days int Days before auto-close
    saml2_enabled boolean SAML2 SSO enabled
    ldap_enabled boolean LDAP authentication enabled

    Primary Key: (org_id)

    SAI Indexes: org_name

    branding_config¶

    Per-organization white-label theme configuration.

    Compaction: LeveledCompactionStrategy

    Column Type Description
    org_id text Organization identifier
    config_key text Configuration key (clustering, ASC)
    app_title text Application title
    primary_color text Primary theme color
    secondary_color text Secondary theme color
    accent_color text Accent color
    background_color text Background color
    surface_color text Surface color
    font_family text Typography font family
    custom_css text Custom CSS (max 64KB, sanitized)

    Primary Key: (org_id, config_key)

    branding_assets¶

    Binary assets for white-label branding (logos, favicons, PWA icons). Max 2MB per asset.

    Compaction: LeveledCompactionStrategy

    Column Type Description
    org_id text Organization identifier
    asset_key text Asset identifier (clustering, ASC)
    data blob Binary asset data
    content_type text MIME type
    file_name text Original file name
    file_size int File size in bytes
    etag text Cache control ETag

    Primary Key: (org_id, asset_key)
