    MCP Tools¶

    The WitFoo MCP server exposes a set of security-focused tools that external AI clients can discover and invoke. Each tool has a defined input schema, executes against live security data, and returns structured results.

    Tool Discovery¶

    AI clients discover available tools by sending a tools/list request:

    {
  "jsonrpc": "2.0",
  "id": 1,
  "method": "tools/list"
}

    The server responds with the complete list of tools and their JSON Schema definitions.

    Available Tools¶

    search_artifacts¶

    Query security events including IDS alerts, firewall logs, authentication events, and DNS queries.

    Input Schema:

    {
  "type": "object",
  "properties": {
    "time_range": {
      "type": "string",
      "description": "Predefined range: last_1h, last_24h, last_7d, last_30d",
      "enum": ["last_1h", "last_24h", "last_7d", "last_30d"]
    },
    "start_time": {
      "type": "string",
      "format": "date-time",
      "description": "ISO 8601 start time (alternative to time_range)"
    },
    "end_time": {
      "type": "string",
      "format": "date-time",
      "description": "ISO 8601 end time (alternative to time_range)"
    },
    "severity": {
      "type": "string",
      "enum": ["low", "medium", "high", "critical"],
      "description": "Filter by severity level"
    },
    "artifact_type": {
      "type": "string",
      "description": "Filter by artifact type (e.g., ids_alert, firewall, auth, dns)"
    },
    "alarm": {
      "type": "boolean",
      "description": "Filter by alarm status"
    },
    "limit": {
      "type": "integer",
      "default": 25,
      "maximum": 100,
      "description": "Maximum number of results"
    }
  }
}

    Example Request:

    {
  "jsonrpc": "2.0",
  "id": 2,
  "method": "tools/call",
  "params": {
    "name": "search_artifacts",
    "arguments": {
      "time_range": "last_24h",
      "severity": "critical",
      "limit": 10
    }
  }
}

    Example Response:

    {
  "jsonrpc": "2.0",
  "id": 2,
  "result": {
    "content": [
      {
        "type": "text",
        "text": "Found 3 critical artifacts in the last 24 hours:\n\n1. [IDS Alert] ET EXPLOIT Possible CVE-2024-1234 - src: 10.0.1.50, dst: 192.168.1.10 - 2025-01-15T08:23:00Z\n2. [Auth] Brute force detected - user: admin, src: 203.0.113.45 - 2025-01-15T06:15:00Z\n3. [Firewall] Blocked outbound C2 - src: 10.0.2.30, dst: 198.51.100.77:443 - 2025-01-15T03:42:00Z"
      }
    ]
  }
}

    search_nodes¶

    Look up network assets, users, domains, and processes in the security knowledge graph.

    Input Schema:

    {
  "type": "object",
  "properties": {
    "query": {
      "type": "string",
      "description": "Search query (IP address, hostname, username, domain)"
    },
    "node_type": {
      "type": "string",
      "enum": ["ip", "hostname", "user", "domain", "process"],
      "description": "Filter by node type"
    },
    "limit": {
      "type": "integer",
      "default": 25,
      "maximum": 100,
      "description": "Maximum number of results"
    }
  },
  "required": ["query"]
}

    Example Request:

    {
  "jsonrpc": "2.0",
  "id": 3,
  "method": "tools/call",
  "params": {
    "name": "search_nodes",
    "arguments": {
      "query": "10.0.1.50",
      "node_type": "ip"
    }
  }
}

    Example Response:

    {
  "jsonrpc": "2.0",
  "id": 3,
  "result": {
    "content": [
      {
        "type": "text",
        "text": "Found node: 10.0.1.50\n- Type: ip\n- Hostname: workstation-042.corp.local\n- First seen: 2024-11-01T00:00:00Z\n- Last seen: 2025-01-15T10:30:00Z\n- Associated artifacts: 147\n- Risk score: 78/100"
      }
    ]
  }
}

    get_incident¶

    Retrieve detailed information about a specific incident.

    Input Schema:

    {
  "type": "object",
  "properties": {
    "incident_id": {
      "type": "string",
      "description": "Incident identifier"
    }
  },
  "required": ["incident_id"]
}

    Example Request:

    {
  "jsonrpc": "2.0",
  "id": 4,
  "method": "tools/call",
  "params": {
    "name": "get_incident",
    "arguments": {
      "incident_id": "inc-2024-0042"
    }
  }
}

    query_graph¶

    Execute queries against the security knowledge graph to discover relationships between entities.

    Input Schema:

    {
  "type": "object",
  "properties": {
    "source": {
      "type": "string",
      "description": "Source entity (IP, hostname, user, domain)"
    },
    "relationship": {
      "type": "string",
      "enum": ["communicates_with", "authenticates_to", "resolves_to", "associated_with"],
      "description": "Relationship type to traverse"
    },
    "depth": {
      "type": "integer",
      "default": 1,
      "maximum": 3,
      "description": "Graph traversal depth"
    },
    "limit": {
      "type": "integer",
      "default": 25,
      "maximum": 100
    }
  },
  "required": ["source"]
}

    Example Request:

    {
  "jsonrpc": "2.0",
  "id": 5,
  "method": "tools/call",
  "params": {
    "name": "query_graph",
    "arguments": {
      "source": "10.0.1.50",
      "relationship": "communicates_with",
      "depth": 2
    }
  }
}

    Tool Permissions¶

    All MCP tools require at minimum the ai:read permission. The permission model is:

    Tool Required Permission
    search_artifacts ai:read
    search_nodes ai:read
    get_incident ai:read
    query_graph ai:read

    Response Format¶

    All tool responses follow the MCP content format:

    {
  "content": [
    {
      "type": "text",
      "text": "Human-readable result text"
    }
  ]
}

    Results are returned as formatted text optimized for LLM consumption. The text includes specific data points (IPs, timestamps, counts) that the AI client can reference in its responses.
