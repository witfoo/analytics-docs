    Graph Nodes¶

    Graph nodes represent unique security entities observed in your environment. Each node is a distinct IP address, domain, user, host, file hash, or other entity extracted from ingested signals.

    Navigate to: Graph → Nodes

    Node Types¶

    Type Icon Description Example
    IP Address Network icon IPv4 or IPv6 address 192.168.1.100, 2001:db8::1
    Domain Globe icon DNS domain name example.com, evil.xyz
    Hostname Server icon System hostname web-server-01, dc01.corp.local
    User Person icon User account jsmith, admin@corp.local
    File Hash Document icon MD5, SHA1, or SHA256 hash a1b2c3d4...
    Email Mail icon Email address user@example.com
    URL Link icon Full URL https://evil.xyz/payload
    Process Gear icon Process name powershell.exe, curl

    Node Properties¶

    Click any node to view its properties panel. Every node includes:

    Core Properties¶

    Property Description
    Node ID Unique identifier
    Type Entity type (IP, domain, user, etc.)
    Value The entity value
    First Seen Timestamp of first observation
    Last Seen Timestamp of most recent observation
    Signal Count Number of signals referencing this entity

    Classification Properties¶

    Property Description
    Managed Whether the entity is under organizational control
    Internal Whether the entity is within the network perimeter
    Trusted Whether the entity is known-good
    Infrastructure Whether the entity is network/security infrastructure
    Frameworks Applied compliance framework tags

    Enrichment Properties¶

    Property Description
    GeoIP Country, city, ASN, organization (IP nodes only)
    Threat Intel Threat category, confidence, source
    ATT&CK Techniques Associated MITRE ATT&CK technique IDs
    DNS Resolved hostname or IP address

    Browsing Nodes¶

    The Nodes page displays a searchable, filterable table of all graph nodes.

    Searching¶

    Use the search bar to find nodes by value:

    192.168.1.100

    evil.xyz

    Filtering¶

    Filter nodes using the controls above the table:

    • Node Type — Select one or more entity types
    • Classification — Filter by managed, internal, trusted, infrastructure
    • Framework — Filter by applied framework tags
    • Time Range — Filter by first/last seen timestamps

    Sorting¶

    Click column headers to sort by:

    • Value (alphabetical)
    • Type
    • First Seen / Last Seen
    • Signal Count

    Node Classification¶

    Nodes are classified by Classification Rules configured in the Signals module. Classifications determine how nodes appear in the graph and how they contribute to compliance reporting.

    Viewing Classification Status¶

    The Classification column in the nodes table shows applied classifications as colored tags:

    Tag Color Classification
    Blue Managed
    Green Internal
    Teal Trusted
    Purple Infrastructure
    Gray Unclassified

    Unclassified Nodes¶

    Nodes without any classification appear with a gray "Unclassified" tag. To improve your security posture visibility:

    1. Filter the nodes table to show only unclassified nodes
    2. Review the entity values to determine appropriate classifications
    3. Create or update Classification Rules to cover these entities

    Prioritize Unclassified Internal IPs

    Start by classifying nodes in your internal IP ranges. Unclassified internal assets represent blind spots in your compliance posture.

    Node Detail Page¶

    Click a node row to open its detail page, which shows:

    1. Properties panel — All core, classification, and enrichment properties
    2. Connected edges — All relationships to other nodes
    3. Signal timeline — Chronological list of signals referencing this entity
    4. Related work units — Observer incidents involving this entity
    5. ATT&CK techniques — Mapped MITRE ATT&CK techniques

    Common Tasks¶

    Investigate a Suspicious IP¶

    1. Navigate to Graph → Nodes
    2. Search for the IP address
    3. Click the node to view its detail page
    4. Check Threat Intel properties for known-bad indicators
    5. Review Connected Edges to see what the IP communicated with
    6. Check Related Work Units for existing incidents

    Find All Unmanaged Assets¶

    1. Navigate to Graph → Nodes
    2. Set the Classification filter to exclude "Managed"
    3. Set the Node Type filter to "IP Address" or "Hostname"
    4. Review the results to identify assets that should be classified

    Export Node Data¶

    Click the Export CSV button to download the current filtered node list as a CSV file for offline analysis or reporting.
