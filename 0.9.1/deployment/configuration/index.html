    Configuration¶

    WitFoo Analytics is configured through environment variables, .env files, and Docker secrets. This reference covers all required and optional variables for each service.

    Environment Variable Precedence¶

    1. Shell environment — Variables exported in the current shell
    2. .env file — Loaded automatically by Docker Compose from the docker/ directory
    3. Compose file defaults — Inline defaults using ${VAR:-default} syntax
    4. Image defaults — Baked into Dockerfiles via ENV directives

    The .env File¶

    Create a .env file in the docker/ directory alongside your compose file:

    # docker/.env

# Secrets (required for production)
JWT_SECRET=your-256-bit-secret-here
WF_LICENSE=your-license-key
AI_ENCRYPTION_KEY=your-ai-encryption-key
AUTH_CONFIG_ENCRYPTION_KEY=your-base64-encoded-32-byte-key

# Cassandra tuning
CASSANDRA_HEAP=6G
CASSANDRA_NEWSIZE=1200M

# Network binding
LISTEN_IP=0.0.0.0

# TLS
TLS_ENABLED=true

# Grafana
GF_ADMIN_PASSWORD=your-grafana-password

# UI modules
VITE_UI_MODULES=all

    Protect the .env File

    The .env file contains secrets such as JWT_SECRET and encryption keys. Set restrictive permissions (chmod 600 .env) and never commit it to version control. Add .env to your .gitignore.

    Global Variables¶

    These variables are shared across multiple services:

    Variable Required Default Description
    JWT_SECRET Yes Dev placeholder JWT signing secret; must be identical across API, incident-engine, dispatcher, and reverse-proxy
    WF_LICENSE No (empty) WitFoo license key
    LOG_LEVEL No info Log verbosity: debug, info, warn, error
    LISTEN_IP No 0.0.0.0 IP address for port bindings; set to 127.0.0.1 to restrict access
    CASSANDRA_KEYSPACE No analytics_v2 Cassandra keyspace name
    NATS_URL No nats://nats:4222 NATS connection URL

    Change JWT_SECRET in Production

    The default value witfoo-analytics-dev-secret-change-in-production is for development only. Generate a cryptographically random secret of at least 256 bits for production deployments.

    Cassandra Variables¶

    Variable Required Default Description
    CASSANDRA_CLUSTER_NAME No WitFoo-Analytics Cluster identifier
    CASSANDRA_DC No dc1 Data center name
    MAX_HEAP_SIZE / CASSANDRA_HEAP No 6G JVM max heap size
    HEAP_NEWSIZE / CASSANDRA_NEWSIZE No 1200M JVM young generation size
    WF_CASSANDRA_PW No cassandra Cassandra password
    CASSANDRA_SEEDS Clustered cassandra Comma-separated seed node IPs
    CASSANDRA_LISTEN_ADDRESS Clustered 0.0.0.0 Address for inter-node communication
    NODE_IP Clustered 0.0.0.0 Node's broadcast address

    Cassandra Heap Sizing

    For CI environments with limited memory, reduce the heap: CASSANDRA_HEAP=2G CASSANDRA_NEWSIZE=400M. Never exceed 50% of available system RAM for the Cassandra heap.

    API Service Variables¶

    Variable Required Default Description
    PORT No 8080 HTTP listen port
    JWT_TOKEN_DURATION No 24h JWT token expiry duration
    AI_ENCRYPTION_KEY No (empty) Encryption key for AI configuration
    AUTH_CONFIG_ENCRYPTION_KEY No (empty) 32-byte base64 key for LDAP/SAML encryption
    API_TTL_NODES No 604800 Graph node cache TTL (seconds)
    API_TTL_EDGES No 604800 Graph edge cache TTL (seconds)
    API_TTL_NODE_INDEX No 604800 Node index cache TTL (seconds)
    WF_CASSANDRA_SEED_NODES No cassandra Cassandra contact points
    WF_CASSANDRA_DATACENTER No dc1 Cassandra data center for connection

    Incident Engine Variables¶

    Variable Required Default Description
    PORT No 8082 HTTP listen port
    CACHE_TTL No 5m In-memory cache TTL (10s for E2E testing)
    MOS_DEFINITIONS_DIR No /meta-definitions/mos Path to MOS rule definitions
    ORG_ID No witfoo Default organization ID
    GIN_MODE No release Gin framework mode (release or debug)

    Graph Processor Variables¶

    Variable Required Default Description
    GRAPH_PROCESSOR_TTL_ARTIFACTS No 2592000 Artifact TTL in seconds (30 days)
    GRAPH_PROCESSOR_TTL_NODES No 604800 Graph node TTL in seconds (7 days)
    GRAPH_PROCESSOR_TTL_EDGES No 604800 Graph edge TTL in seconds (7 days)
    GRAPH_PROCESSOR_TTL_NODE_INDEX No 604800 Node index TTL in seconds (7 days)
    GRAPH_PROCESSOR_TTL_LEAD_RULES No 86400 Lead rules cache TTL in seconds (24 hours)
    CASSANDRA_HOSTS No cassandra:9042 Cassandra host:port pairs

    Reverse Proxy Variables¶

    Variable Required Default Description
    API_URL No http://api:8080 Backend API URL
    UI_URL No http://ui:3000 Frontend UI URL
    DISPATCHER_URL No http://dispatcher:8003 Dispatcher service URL
    INCIDENT_ENGINE_URL No http://incident-engine:8082 Incident engine URL
    TLS_ENABLED No true Enable HTTPS termination
    TLS_PORT No 443 HTTPS listen port
    CERT_FILE No ./certs/server.crt Path to TLS certificate
    KEY_FILE No ./certs/server.key Path to TLS private key

    UI Variables¶

    Variable Required Default Description
    PORT No 3000 HTTP listen port
    PUBLIC_BASE_PATH No /ui URL base path
    API_BASE_URL No http://api:8080 Backend API URL (server-side)
    VITE_UI_MODULES No all UI module visibility (build-time)
    ORIGIN No http://localhost:3000 Allowed CORS origin

    UI Module Options¶

    Value Modules Enabled
    all Search, Observer, Reporter, CyberGrid, AI
    search_only Search
    search_observer Search, Observer

    Build-Time Variable

    VITE_UI_MODULES is a build-time argument. Changing it requires rebuilding the UI container image.

    Observability Variables¶

    Variable Required Default Description
    GF_ADMIN_PASSWORD No admin Grafana admin password
    GRAFANA_REMOTE_WRITE_URL No (empty) Remote write endpoint for Grafana Cloud
    GRAFANA_REMOTE_WRITE_USER No (empty) Remote write username
    GRAFANA_REMOTE_WRITE_API_KEY No (empty) Remote write API key

    Docker Secrets¶

    For production deployments, use Docker secrets instead of environment variables for sensitive values:

    services:
  api:
    secrets:
      - jwt_secret

secrets:
  jwt_secret:
    file: ./secrets/jwt_secret.txt

    Create the secrets directory:

    mkdir -p docker/secrets
chmod 700 docker/secrets

# Generate a random JWT secret
openssl rand -base64 32 > docker/secrets/jwt_secret.txt
chmod 600 docker/secrets/jwt_secret.txt

    Retention Configuration¶

    Data retention is configured via the Onboarding Wizard in the Analytics web UI after initial deployment. Default values:

    Setting Environment Variable Default
    Artifact retention RETENTION_ARTIFACTS_DAYS 7 days
    Work unit retention RETENTION_WORK_UNITS_DAYS 365 days
    Work collection retention RETENTION_WORK_COLLECTIONS_DAYS 365 days
    Report retention RETENTION_REPORTS_DAYS 1825 days (5 years)

    Configuration Files¶

    WFA production deployments generate JSON configuration files:

    /witfoo/configs/analytics.json:

    {
  "role": "analytics-aio",
  "retention": {
    "artifacts_days": 7,
    "work_units_days": 365,
    "work_collections_days": 365,
    "reports_days": 1825
  },
  "ui_modules": "all",
  "cybergrid": {
    "enabled": false,
    "publish_enabled": false
  },
  "clustering": {}
}

    See WFA Integration for details on the configuration wizard.
