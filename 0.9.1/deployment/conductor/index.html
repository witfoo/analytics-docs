    Conductor Service Deployment¶

    The Conductor suite is WitFoo's signal collection pipeline. It receives raw syslog, Beats, and TLS-encrypted log data, parses signals into structured artifacts, and exports them to the Analytics artifact-ingestion service. Conductor services are included in the AIO + Conductor role or can be deployed standalone alongside an existing Analytics stack.

    Signal Pipeline¶

    graph LR
    SYSLOG[Syslog Sources<br>UDP/TCP 514] --> SS[Signal Server]
    TLS_SRC[TLS Sources<br>:6514] --> SS
    BEATS[Beats Agents<br>:5044] --> SS
    SS -->|raw signals| BE[Broker Edge<br>NATS :4223]
    BE --> SP[Signal Parser]
    SP -->|parsed artifacts| BE
    BE --> AF[Artifact Filter]
    AF -->|enriched artifacts| BE
    BE --> AE[Artifact Exporter]
    AE -->|HTTP POST| AI[Analytics<br>Artifact Ingestion :8090]

    Conductor Services¶

    Service Container Ports Description
    Broker Edge conductor-broker-edge 4223 (NATS), 8223 (monitoring) Dedicated NATS instance for Conductor message routing
    Signal Server conductor-signal-server 514/udp, 514/tcp, 5044, 6514 Receives syslog (UDP/TCP), Beats, and TLS-encrypted signals
    Signal Parser conductor-signal-parser — Parses raw signals into structured artifacts
    Artifact Filter conductor-artifact-filter — Filters and enriches artifacts (STIX, threat intelligence)
    Artifact Exporter conductor-artifact-exporter — Exports finalized artifacts to Analytics via HTTP
    Conductor UI conductor-ui 8000 Web management interface

    NATS Topology¶

    Conductor uses its own NATS instance (Broker Edge) on port 4223, separate from the Analytics NATS on port 4222. Both share the witfoo-analytics-net Docker network.

    graph TB
    subgraph Analytics NATS :4222
        AI_NATS[nats]
        AI[artifact-ingestion] --> AI_NATS
        GP[graph-processor] --> AI_NATS
        IE[incident-engine] --> AI_NATS
        DISP[dispatcher] --> AI_NATS
    end

    subgraph Conductor NATS :4223
        BE[broker-edge]
        SS[signal-server] --> BE
        SP[signal-parser] --> BE
        AF[artifact-filter] --> BE
        AE[artifact-exporter] --> BE
    end

    AE -->|HTTP| AI

    Standalone Deployment¶

    If your Analytics stack is already running, deploy Conductor services separately using the standalone compose file:

    Prerequisites¶

    The Analytics network must already exist:

    # Verify the network exists
docker network ls | grep witfoo-analytics-net

    Build and Start¶

    # Build Conductor binaries
./scripts/conductor-build.sh all

# Build container images
docker compose -f docker-compose-conductor.yml build

# Start Conductor services
docker compose -f docker-compose-conductor.yml up -d

    Standalone Compose Configuration¶

    The standalone compose file joins the existing Analytics network:

    networks:
  witfoo-analytics-net:
    external: true

services:
  conductor-broker-edge:
    build:
      context: ../conductor_suite/broker-edge
    container_name: conductor-broker-edge-svc
    ports:
      - "4223:4223"
      - "8223:8223"
    environment:
      - WF_NODE_CONFIG_FILE=/configs/node.json
    volumes:
      - ./conductor-node.json:/configs/node.json:ro
    networks:
      - witfoo-analytics-net
    healthcheck:
      test: ["CMD", "nats", "server", "ping", "--server=nats://127.0.0.1:4223"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 15s

    AIO + Conductor Deployment¶

    For single-node deployments, use the combined compose file:

    docker compose -f docker-compose.aio-conductor.yml up -d --build

    This deploys all 15 services (11 Analytics + 6 Conductor) on a single witfoo-analytics-net bridge network. The Conductor services are defined in the same compose file:

    # Conductor services in docker-compose.aio-conductor.yml

broker-edge:
  build:
    context: ../conductor_suite/broker-edge
  container_name: conductor-broker-edge
  ports:
    - "4223:4223"
    - "8223:8223"
  volumes:
    - ../conductor_suite/test/configs:/configs:ro
    - ../conductor_suite/test/certs:/certs:ro
    - conductor-nats-data:/data/nats
  environment:
    - WF_NODE_CONFIG_FILE=/configs/node.json
  networks:
    witfoo-analytics-net:
      aliases:
        - broker-edge.lan

artifact-exporter:
  build:
    context: ../conductor_suite/artifact-exporter
  container_name: conductor-artifact-exporter
  environment:
    - WF_NODE_CONFIG_FILE=/configs/node.json
    - ANALYTICS_INGESTION_URL=http://artifact-ingestion:8090
    - ANALYTICS_SECRET=${ANALYTICS_SECRET:-}
  depends_on:
    broker-edge:
      condition: service_healthy
    artifact-ingestion:
      condition: service_started

    Configuration¶

    Node Configuration¶

    All Conductor services read from a shared node.json configuration file:

    {
  "node_id": "conductor-01",
  "nats_url": "nats://broker-edge:4223",
  "analytics_api_url": "http://reverse-proxy:8081",
  "certs_dir": "/certs"
}

    Mount this file as a read-only volume:

    volumes:
  - ./conductor-node.json:/configs/node.json:ro

    Environment Variables¶

    Variable Service Default Description
    WF_NODE_CONFIG_FILE All Conductor /configs/node.json Path to node configuration
    ANALYTICS_API_URL signal-parser, conductor-ui http://reverse-proxy:8081 Analytics API endpoint
    ANALYTICS_INGESTION_URL artifact-exporter http://artifact-ingestion:8090 Artifact ingestion endpoint
    ANALYTICS_SECRET artifact-exporter (empty) Shared secret for artifact delivery
    DEBUG artifact-exporter 0 Enable debug logging
    DEBUG_MODULES artifact-exporter (empty) Comma-separated debug modules

    Shared Secret

    If ANALYTICS_SECRET is set on the artifact-ingestion service, the same value must be configured on the artifact-exporter. Mismatched secrets cause artifact delivery failures.

    Signal Server Ports¶

    The Signal Server listens on multiple ports for different log transport protocols:

    Port Protocol Description
    514/udp Syslog UDP Standard syslog (RFC 5424)
    514/tcp Syslog TCP Reliable syslog delivery
    1514 Syslog TCP Alternative syslog port (non-privileged)
    5044 Beats Elastic Beats protocol (Filebeat, Winlogbeat)
    6514 Syslog TLS Encrypted syslog (RFC 5425)

    Privileged Ports

    Ports 514 require the NET_BIND_SERVICE capability. This is configured in the compose file via cap_add. On SELinux-enabled systems, additional policy configuration may be required.

    signal-server:
  ports:
    - "514:514/udp"
    - "514:514/tcp"
    - "5044:5044"
    - "6514:6514"
  cap_add:
    - NET_BIND_SERVICE

    Volumes¶

    Volume Service Purpose
    conductor-nats-data broker-edge NATS JetStream persistence
    conductor-exporters-data artifact-exporter Export state and retry queue
    conductor-integrations-data — Integration configuration storage

    Health Checks¶

    The Broker Edge service is the dependency anchor for all other Conductor services:

    conductor-signal-server:
  depends_on:
    conductor-broker-edge:
      condition: service_healthy

    Verify Conductor health:

    # Check Broker Edge NATS
curl http://localhost:8223/healthz

# Check all Conductor containers
docker ps --filter "name=conductor-" --format "table {{.Names}}\t{{.Status}}"

    Troubleshooting¶

    Artifact Exporter Not Delivering¶

    # Check exporter logs
docker logs conductor-artifact-exporter --tail=50

# Verify connectivity to artifact-ingestion
docker exec conductor-artifact-exporter wget -q -O- http://artifact-ingestion:8090/health

    Signal Server Not Receiving¶

    # Test syslog delivery
echo "<14>Test message from deployment verification" | nc -u localhost 514

# Check signal-server logs
docker logs conductor-signal-server --tail=50

    Broker Edge Connection Failures¶

    # Verify NATS is healthy
docker exec conductor-broker-edge nats server ping --server=nats://127.0.0.1:4223

# Check NATS monitoring
curl http://localhost:8223/varz
